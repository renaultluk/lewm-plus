# AGENTS Guide

## Project Overview
- This repo contains LeWorldModel (LeWM): a JEPA-style world model trained end-to-end from pixels.
- Core entry points are `train.py` (training) and `eval.py` (planning/evaluation).
- Hydra drives configuration from `config/train/` and `config/eval/`.
- The project relies on `stable-worldmodel` (data/env/planning) and `stable-pretraining` (training loop/module wrappers).
- Custom environments and dataset generators live in `scripts/`.

## Key Layout
- `train.py`, `eval.py`: main scripts.
- `jepa.py`, `module.py`, `utils.py`: model/loss/support code.
- `scripts/reacher_custom_env.py`: custom Reacher env with `qpos_match`/`reach`/`push` task modes.
- `scripts/generate_reacher_custom_eval_dataset.py`: generates HDF5 eval datasets for any task mode.
- `scripts/generate_mujoco_dataset.py`: generates Lance training datasets (random or multitask expert).
- `config/train/lewm.yaml`: base train config; data profiles in `config/train/data/`.
- `config/eval/*.yaml`: eval task configs (`reacher_reach.yaml`, `reacher_push.yaml`, `pusht.yaml`, etc.).
- `superpod/`: HKUST SuperPOD helpers (self-submitting Slurm scripts + container tooling).
- `docs/SUPERPOD_GUIDE.md`: SuperPOD runbook.

## Dev Environment Tips

### Local
```bash
uv venv --python=3.10
source .venv/bin/activate
uv pip install stable-worldmodel[train,env]
```

### Runtime data location
- `STABLEWM_HOME` controls where datasets/checkpoints are resolved.
- Default is `~/.stable-wm` unless overridden.
- In SuperPOD jobs, scripts mount host `${STABLEWM_HOME}` into container at `/workspace/.stable-wm`.
- Must pass `--container-env STABLEWM_HOME=/workspace/.stable-wm` to `srun` because the Dockerfile env var is not reliably propagated.

### Environment Setup
- The custom env `swm/ReacherCustom-v0` (registered by `scripts/reacher_custom_env.py`) wraps `ReacherEnv` with the custom XML `assets/reacher_task_agnostic.xml`.
- All scripts that import `scripts.reacher_custom_env` need the repo root on `sys.path` (handled automatically by `generate_reacher_custom_eval_dataset.py`).
- EGL context must be created at module top (before any MuJoCo import) for headless rendering. This is handled by all dataset generators.

## Reacher Environment

### Custom XML (`assets/reacher_task_agnostic.xml`)
10-DOF MuJoCo scene with:
- qpos[0:2]: arm joints (joint0, joint1) — the 2-DOF robot arm
- qpos[2:4]: target body (red sphere, static)
- qpos[4:6]: blue_goal (target for push)
- qpos[6:8]: blue_object (pushable object)
- qpos[8:10]: purple_ball (another pushable object)

Wrap: `ReacherCustomEvalEnv` in `scripts/reacher_custom_env.py`, registered as `swm/ReacherCustom-v0`.

### Task Modes
The env accepts a `task` parameter:

| Mode | Callable arg | Success condition | Threshold |
|------|-------------|-------------------|-----------|
| `qpos_match` | `target_qpos[:2]` | `\|arm_qpos - target_qpos[:2]\| < 0.05` | All dims |
| `reach` | `target_qpos[2:4]` | `\|fingertip_xy - target_xy\| < 0.04` | 2D dist |
| `push` | `target_qpos[4:6]` | `\|blue_object_xy - blue_goal_xy\| < 0.025` | 2D dist |

The callable `set_target_qpos(target_qpos)` receives the full future qpos vector (10-dim) and extracts the relevant sub-vector based on the current task mode.

## Datasets

### Eval Datasets (HDF5)
Generated by `scripts/generate_reacher_custom_eval_dataset.py`:
```bash
python scripts/generate_reacher_custom_eval_dataset.py --task reach --episodes 50 --max-steps 100
python scripts/generate_reacher_custom_eval_dataset.py --task push --episodes 50 --max-steps 100
```
Stores: `pixels`, `action`, `qpos`, `qvel`, `episode_idx`, `step_idx`.
Default output: `<STABLEWM_HOME>/datasets/reacher_<task>_eval.h5`.

### Training Datasets (Lance)
Two types per task, generated by `scripts/generate_mujoco_dataset.py`:

| Dataset | Policy | Episodes | Steps | Format |
|---------|--------|----------|-------|--------|
| `reacher_reach_random.lance` | Random | 500 | 1000 | Lance |
| `reacher_reach_expert.lance` | multitask (`reach_red_spot`) | 200 | 100 | Lance |
| `reacher_push_random.lance` | Random | 500 | 1000 | Lance |
| `reacher_push_expert.lance` | multitask (`push_blue_object_to_blue_spot`) | 200 | 100 | Lance |

Random data uses the `ReacherTaskAgnostic-v0` env (Reacher-v5 with custom XML) with a uniform random policy. Expert data uses `--policy reacher_multitask` with task-specific controllers.

The convenience script `scripts/generate_all_datasets.sh` generates all datasets in one job.

All generators pass `width=height=image_size` to the env constructor so training and eval pixels are rendered at the same native resolution (224×224).

## Training

Two-stage training is the primary workflow: pretrain on random data (general dynamics), then finetune on expert data (task-specific).

### Task-Specific Training

**Reach:**

```bash
# Phase 1 — Pretrain on random exploration
python train.py data=reacher_reach_random experiment=pretrain \
  trainer.max_epochs=50 output_model_name=reacher_reach_pretrain

# Phase 2 — Finetune on reach expert demos
python train.py data=reacher_reach_expert experiment=finetune \
  trainer.max_epochs=100 output_model_name=reacher_reach_finetuned \
  ckpt_path=/path/to/reacher_reach_pretrain_weights.ckpt
```

**Push:**

```bash
# Phase 1 — Pretrain
python train.py data=reacher_push_random experiment=pretrain \
  trainer.max_epochs=50 output_model_name=reacher_push_pretrain

# Phase 2 — Finetune
python train.py data=reacher_push_expert experiment=finetune \
  trainer.max_epochs=100 output_model_name=reacher_push_finetuned \
  ckpt_path=/path/to/reacher_push_pretrain_weights.ckpt
```

### SuperPOD
```bash
bash superpod/train_lewm.sh data=reacher_reach_random experiment=pretrain \
  trainer.max_epochs=50 output_model_name=reacher_reach_pretrain wandb.enabled=false

bash superpod/train_lewm.sh data=reacher_reach_expert experiment=finetune \
  trainer.max_epochs=100 output_model_name=reacher_reach_finetuned \
  ckpt_path=/workspace/.stable-wm/checkpoints/reacher_reach_pretrain/reacher_reach_pretrain_weights.ckpt \
  wandb.enabled=false
```

Key flags:
- `sigreg_enabled=false` — disables SIGReg loss (only prediction loss remains, used in finetuning).
- `freeze_encoder=true` — sets `requires_grad=False` on all encoder parameters (used in finetuning).
- `ckpt_path=/path/to/model.ckpt` — loads pretrained weights.
- `experiment=pretrain` / `experiment=finetune` — sets the flags above via config profiles.

## Evaluation

Task-specific eval configs: `config/eval/reacher_reach.yaml`, `config/eval/reacher_push.yaml`.

```bash
# Local
python eval.py --config-name=reacher_reach.yaml policy=random eval.num_eval=2
python eval.py --config-name=reacher_reach.yaml policy=reacher_reach_finetuned/weights_epoch_100.pt eval.num_eval=50

# SuperPOD
bash superpod/evaluate_lewm.sh --config-name=reacher_reach.yaml \
  policy=reacher_reach_finetuned/weights_epoch_100.pt eval.num_eval=50 \
  cache_dir=/workspace/.stable-wm
```

Eval configs use `callables` to invoke `set_state` (positions all bodies from dataset qpos) and `set_target_qpos` (extracts task-specific goal from future qpos).

```bash
    --container-writable --container-env=STABLEWM_HOME=/workspace/.stable-wm \
```

```bash

python eval.py --config-name=reacher_reach.yaml policy=random eval.num_eval=2
```


## Testing and Verification
- There is currently no formal `pytest` suite in this repo.
- Minimum checks before pushing:
  - Shell syntax for helper scripts:
    ```bash
    bash -n superpod/*.sh
    ```
  - Python import smoke test:
    ```bash
    cd /path/to/repo && PYTHONPATH=. .venv/bin/python -c "import stable_worldmodel, stable_pretraining, scripts.reacher_custom_env"
    ```
  - If touching configs/scripts, run a short train smoke (`max_epochs=1`) or a random-policy eval (`eval.num_eval=2`).
- For SuperPOD changes, always validate both:
  - CPU hello job: `bash superpod/hello_superpod.sh`
  - GPU/container hello job: `bash superpod/hello_superpod_gpu.sh`

## Contribution Notes
- Keep `superpod/superpod.env` out of git (contains user/group specifics).
- Large artifacts are intentionally ignored (`.stable-wm/`, `*.h5`, `*.ckpt`, `outputs/`, etc.).
- Prefer Hydra overrides for experiment changes instead of hard-coding values.
- Keep SuperPOD scripts self-submitting and source `superpod/_common.sh` for shared env handling.

## Troubleshooting Quick Hits
- `ModuleNotFoundError: No module named 'scripts'`: the repo root must be on `sys.path`. Use `PYTHONPATH=/path/to/repo` or the built-in sys.path fix in the script.
- `Cannot resolve 'reacher_reach_random'`: check that the dataset exists under `${STABLEWM_HOME}/datasets/`.
- `sbatch/srun` module warning on SuperPOD: ensure scripts load `slurm` module before submission/runtime.
- Pyxis/container issues: verify `CONTAINER_PATH` in `superpod/superpod.env` and pass `--container-env STABLEWM_HOME=/workspace/.stable-wm`.
- GPU quota exceeded (`AssocGrpGRESMinutes`): check `squota` and wait for reset, or reduce requested time.
- CPU partition: containers do not work reliably on `cpu`; use the GPU (`normal`) partition even for dataset generation.
- Render resolution: training data must be rendered at the same native resolution as eval data. The `make_env()` function now passes `width=image_size, height=image_size` to `gym.make()`.

## Current SuperPOD Status
- Repo: `renaultluk/lewm-plus` with SuperPOD helpers under repo-root `superpod/`.
- Container expectation: `superpod/Dockerfile` sets `STABLEWM_HOME=/workspace/.stable-wm` and uses `/workspace/.venv/bin/python`.
- Mount strategy: host `${STABLEWM_HOME}` is mounted to container `/workspace/.stable-wm`.
- Container env: the Dockerfile sets `STABLEWM_HOME=/workspace/.stable-wm`, but `srun --container-image` does not reliably propagate it. Pass `--container-env STABLEWM_HOME=/workspace/.stable-wm` or use `/usr/bin/env` prefix in practice.
- Walltime control: set `TRAIN_TIME` in `superpod/superpod.env` (e.g. `71:00:00` under a 72h cap). Note: GPU quota may limit actual wall time.
- Training resource controls: set `TRAIN_NODES`, `TRAIN_NTASKS`, and `TRAIN_GPUS_PER_NODE` in `superpod/superpod.env`.
- Checkpoint layout: both `.pt` and `.ckpt` files for a run live under `${STABLEWM_HOME}/checkpoints/<subdir>/`.
- For old runs, use `bash superpod/migrate_checkpoints.sh <run_id> <output_model_name>` to consolidate mixed legacy paths into `${STABLEWM_HOME}/checkpoints/<subdir>/`.
- Preferred training command:

  ```bash
    trainer.max_epochs=50 output_model_name=reacher_reach_pretrain wandb.enabled=false
  ```
